## 二、备课流程

### 教师唯一操作：更新学生档案

教师每周只需更新 `Student-Profiles/profiles/{学生名}.md`，填写：
- 上次课反馈
- 本周薄弱点变化
- 需要调整的内容

### AI 执行流程

```
1. 读取 Student-Profiles/profiles/{学生名}.md
   → 获取学生年级、科目、当前进度、薄弱点、上次反馈

2. 读取 Teaching-Calendar/src/{学段}/{年级}/{科目}/{学期}.md
   → 确认学校当前周次和教学进度

3. 对比学生进度和学校进度
   → 判断：跟上/超前/落后
   → 确定本周教学重点

4. 读取 Teaching-Materials/备课规范.md
   → 按规范生成材料

5. 生成以下文件到 Teaching-Materials/{学生名}/：
   学生/{周次}/
   ├── {编号}-{科目}-{内容}（原卷）.md     ← 打印给学生
   └── {编号}-{科目}-{内容}要点.md         ← 打印给学生
   教师/{周次}/
   ├── {编号}-{科目}-{内容}（解析）.md     ← 教师参考
   └── {编号}-{科目}-{内容}（教案）.md     ← 逐字稿

6. 更新 Teaching-Materials/README.md
   → 同步目录结构和课表
```

## 三、生成材料的核心原则

### 逐字稿（教案）必须做到

1. **打开就能讲** — 口语化大白话，不用教案腔
2. **对应要点文件** — 每个环节标注 `→ 对应 XX 文件位置`
3. **分支判断** — 学生做对/做错分别怎么应对
4. **时间精确** — 精确到分钟，总时长 ≤120 分钟
5. **课前准备清单** — 列出要打印的文件（精确文件名）

### 小测（原卷）必须做到

1. **零答案** — 不能出现任何答案或提示
2. **打印友好** — 顶部写明姓名、日期、限时、满分
3. **难度分层** — 基础 + 理解 + 综合，贴合该学生水平

### 解析必须做到

1. **逐题详解** — 标准答案 + 为什么 + 不选什么 + 考点
2. **评分标准** — 简答题每点标明分值
3. **考点汇总** — 篇末附考点汇总表

## 四、每周备课触发方式

教师说类似以下任一指令时，AI 开始执行备课流程：

- "帮我备 {学生名} 这周的课"
- "{学生名} 这周上什么"
- "生成 {学生名} 第X周材料"

### AI 收到指令后

1. 确认学生名 → 读取对应档案
2. 确认当前周次 → 如教师未指定，从日历推断
3. 按上述流程生成全部材料
4. 输出打印清单（教师只需打印即可）

---

## 五、注意事项

- 南昌市校历：期中约第10-11周，期末约第19-20周
- 九年级和高三下学期为冲刺节奏，非新课
- 每个学生三科共用约2小时，每科约40分钟
- 学生档案是**唯一输入**，日历和规范是**只读参考**
- 生成材料后提醒教师检查并确认
