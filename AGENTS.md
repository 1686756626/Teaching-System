## 二、🔴 全局红线（绝对不可违反）

> **以下红线来自多个仓库的实际约束，违反任何一个都会导致系统故障。**
> 修改任何代码/配置前，必须逐条对照检查。

### 代码红线

| # | 红线 | 所在位置 | 违反后果 |
|---|------|----------|----------|
| R1 | `parseMaterialsTree()` 中 `parts.length === 4` | Dashboard/app.js | Materials 目录结构解析失效，学生列表为空 |
| R2 | `CONFIG.materials.branch` 必须为 `master` | Dashboard/app.js | 无法拉取备课材料 |
| R3 | `CONFIG.calendar.branch` 必须为 `main` | Dashboard/app.js | 无法拉取教学日历 |
| R4 | `CONFIG.profiles.branch` 必须为 `main` | Dashboard/app.js | 无法拉取学生档案 |
| R5 | `fetchFile()` 中 Contents API 的 base64 必须用 `TextDecoder('utf-8')` 解码 | Dashboard/app.js | 中文文件内容乱码或加载失败 |

### Nginx 红线

| # | 红线 | 违反后果 |
|---|------|----------|
| N1 | 不得添加 `gzip on` | JS 文件经 gzip 后浏览器解压失败，全站白屏 |
| N2 | 不得修改 `/gh-api/` 的 `proxy_pass` 地址 | GitHub API 代理失效 |
| N3 | 不得修改 `/gh-raw/` 的 `proxy_pass` 地址 | 文件下载代理失效 |
| N4 | `/gh-api/` 必须携带 `Authorization` header | 私有仓库（Student-Profiles）返回 404 |

### 文件结构红线

| # | 红线 | 所在仓库 | 违反后果 |
|---|------|----------|----------|
| F1 | Materials 目录必须严格 4 层：`学生名/角色/周次/文件名` | Teaching-Materials | 网站解析失效 |
| F2 | 周次必须两位纯数字：`08`（不是 `第8周`） | Teaching-Materials | 网站解析失效 |
| F3 | 角色层只能是 `学生` 或 `教师` | Teaching-Materials | 网站解析失效 |
| F4 | 不可在周次和文件名之间增加子文件夹 | Teaching-Materials | 网站解析失效 |

### 库依赖红线

| # | 红线 | 违反后果 |
|---|------|----------|
| L1 | 所有第三方库必须放在本地 `lib/` 目录 | 国内 CDN 不稳定，改回 CDN 会导致样式/功能丢失 |
| L2 | 不得删除或重命名 `lib/` 目录下的任何文件 | 页面功能异常 |

### GitHub API 操作红线

| # | 红线 | 违反后果 |
|---|------|----------|
| A1 | **不得用 Contents API（`PUT /repos/:owner/:repo/contents/:path`）推送含中文内容的文件** | API 的 base64 编码对中文处理不稳定，文件可能变为 0 字节（2026-04-03 实际发生：Teaching-System/AGENTS.md 被 ee02867 写入 0 字节） |
| A2 | 推送文件**必须**用 `git clone + 本地编辑 + git commit + git push` | 确保文件内容完整，尤其是中文文件名和中文内容 |

## 三、备课流程（严格按顺序执行）

### 步骤 1：读取学生档案

从 `Student-Profiles` 仓库读取：
- `students.json` — 所有学生基本信息（姓名、年级、科目、上课时段、当前周次）
- `profiles/{学生名}.md` — 详细档案（当前进度、薄弱点、上次反馈、学习风格、教学策略）

> 如无法访问私有仓库，请要求教师手动提供学生信息。

### 步骤 2：读取教学日历

从 `Teaching-Calendar` 仓库读取：
- `校历总览.md` — 学期起始日期、假期安排、考试时间
- `src/{学段}/{年级}/{科目}/{学期}.md` — 各科详细教学进度（**这是唯一数据源，不要读 schedule.json**）

确认学校当前周次和教学进度。

### 步骤 3：确定本周教学重点

对比学生进度 vs 学校进度：

| 情况 | 应对策略 |
|------|----------|
| 跟上 | 按学校进度走，巩固 + 适当抢进度 |
| 落后 | 优先补薄弱点，少量抢进度 |
| 超前 | 拓展拔高 + 真题训练 |

### 步骤 4：生成全部材料

> ⚠️ 生成材料前，**必须先读取 `Teaching-Materials/备课规范.md`**，了解学生版/教师版的详细格式要求。

每科每节课只产出 **2 份文档**，按规范写入 `Teaching-Materials/{学生名}/`：

```
{学生名}/
├── 学生/{周次}/
│   └── {编号}-{科目}-{内容}（学生版）.md     ← 学案（打印给学生）
└── 教师/{周次}/
    └── {编号}-{科目}-{内容}（教师版）.md     ← 时间轴操作手册
```

| 文档类型 | 定位 | 说明 |
|----------|------|------|
| **学生版（学案）** | 好的题目 + 要点总结 | 打印给学生做题，课后带走当复习资料，**不含答案** |
| **教师版（操作手册）** | 以时间为轴的操作手册 | 题目+答案+解析+逐字稿+分支判断全部融入时间轴，老师只需这一份 |

> 📌 **标杆文件**：`杨紫泠/教师/08/01-语文-期中备考冲刺（教师版）.md`

### 步骤 5：更新学生档案

如果本次课有值得记录的情况（测验结果、薄弱点发现、家长反馈等），更新 `Student-Profiles/profiles/{学生名}.md` 和 `students.json` 的 `current_week`。

### 步骤 6：输出打印清单

给教师一个清晰的清单：
- 哪些文件需要打印（精确文件名）
- 课表概览（时间 - 科目 - 内容）

## 四、文件命名规范

```
{编号}-{科目}-{内容}（学生版）.md
{编号}-{科目}-{内容}（教师版）.md
```

| 元素 | 说明 | 示例 |
|------|------|------|
| 编号 | 两位数 01-99，按上课顺序 | `01`、`02`、`03` |
| 科目 | 中文简称 | `语文`、`道法`、`历史` |
| 内容 | 具体知识点或本课主题 | `期中备考冲刺`、`社会规则与责任` |
| 类型 | 括号包裹 | `（学生版）`、`（教师版）` |

### 科目名称（重要！）

| 学段 | 日历文件名 | 材料文件名 | 说明 |
|------|-----------|-----------|------|
| 初中 | `道德与法治` | **`道法`** | 文件名必须用简称 |
| 高中 | `思想政治` | **`政治`** | 文件名必须用简称 |
| 通用 | `语文` | `语文` | — |
| 通用 | `历史` | `历史` | — |

## 六、🔄 文档同步规则

> **这是给 AI 智能体的强制性规则。任何接手本项目的 AI 都必须遵守。**

### 规则 1：修改代码 → 同步文档

任何对代码、配置、目录结构、数据格式的修改，**必须同步更新所有相关 AGENTS.md**。
具体来说：

| 你改了什么 | 必须更新的文档 |
|-----------|---------------|
| app.js 中的数据结构或解析逻辑 | Teaching-Dashboard/AGENTS.md |
| app.js 中的 CONFIG 或红线 | Teaching-Dashboard/AGENTS.md + 本文件（红线清单） |
| Teaching-Materials 目录规范 | Teaching-Materials/AGENTS.md + 本文件（红线清单） |
| Teaching-Calendar 数据源 | Teaching-Calendar/AGENTS.md + 本文件（步骤2） |
| Nginx 配置 | Teaching-Dashboard/AGENTS.md + 本文件（红线清单） |
| Student-Profiles 的字段结构 | Student-Profiles/AGENTS.md（如有） + 本文件（步骤1） |

### 规则 2：更新文档 → 追加变更日志

每次修改后，**必须在本文件"变更日志"章节追加一条记录**，格式：
```
[日期] [改动仓库] 改动摘要 + 影响范围
```

### 规则 3：接手项目时 → 先读日志

任何新 AI 接手本项目时，**必须先读完以下内容**，按顺序：

1. 本文件（Teaching-System/AGENTS.md）— 全局规则 + 红线 + 变更日志
2. 各仓库 AGENTS.md — 各自的技术细节
3. 代码实际实现 — 以代码为准，文档如有出入以代码为准

### 规则 4：发现不一致 → 立即修正

如果发现代码和文档不一致，**以代码为准，立即更新文档**，并在变更日志中记录。
