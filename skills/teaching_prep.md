# teaching_prep — 备课全流程技能

> 当用户要求"备课"、"给XX出题"、"生成材料"、"出作业"、"出讲义"时触发本技能。

## 触发条件

- 用户提到"备课"、"出题"、"生成材料"
- 用户提到某个学生名字 + 需要准备内容
- 用户要求出周中作业或周末讲义

## 前置检查：每次备课前必读

1. **Teaching-System/AGENTS.md** — 总规则（红线清单）
2. **Student-Profiles/students.json** — 课表和学生信息
3. **Student-Profiles/profiles/{学生名}.md** — 具体学生档案
4. **Teaching-Calendar/src/{学段}/{年级}/{科目}/{学期}.md** — 学校教学进度
5. **Teaching-Materials/备课规范.md** — 材料格式规范

## 场景 A：出周中作业

### 说明
- 同年级同一份，不按学生区分
- 语文由 AI 出，道法历史由老师从学科网下载
- 每周 4 份（周一到周四）

### 操作步骤

```
1. 确定年级 + 当前周次
2. 读取 Calendar 确认学校进度
3. 确定下次考试范围
4. 按备课规范生成作业
5. 写入 Teaching-Materials：
   路径：{年级}/作业/W{周次}/周X-科目.md
   例：九年级/作业/W09/周一-语文.md
6. git clone + commit + push（禁止用 Contents API）
```

### 作业格式
- 题目围绕下次考试
- 含选择题 + 简答题
- 附评分标准（简答题）

## 场景 B：出周末讲义

### 说明
- 根据电话记录中的错题和薄弱点定制
- 每个学生单独出
- 只讲有问题的，不面面俱到

### 操作步骤

```
1. 读取该学生的 profiles/{学生名}.md
2. 读取电话记录（在学生档案中）
3. 读取 Calendar 确认进度
4. 按备课规范生成讲义
5. 写入 Teaching-Materials：
   路径：{年级}/周末/W{周次}-讲义.md
   例：九年级/周末/W09-讲义.md
6. git clone + commit + push
```

## 场景 C：旧结构备课（学生版+教师版）

### 说明
- 旧结构仍有学生使用，按需生成
- 每科产出 2 份：学生版（学案）+ 教师版（操作手册）

### 操作步骤

```
1. 读取学生档案 + Calendar + 备课规范
2. 确定教学重点（对比学生进度 vs 学校进度）
3. 生成学生版（学案：题目+要点总结，不含答案）
4. 生成教师版（时间轴操作手册：含逐字稿+分支判断）
5. 写入 Teaching-Materials：
   路径：{学生名}/学生/{周次}/{编号}-{科目}-{内容}（学生版）.md
   路径：{学生名}/教师/{周次}/{编号}-{科目}-{内容}（教师版）.md
6. 更新 Materials/README.md
7. 更新 Student-Profiles 学生档案
8. 输出打印清单
9. git clone + commit + push（每个仓库分别操作）
```

### 教师版格式要点
- 头部：学生、日期、时长、学校进度、辅导定位、薄弱点聚焦
- 时间轴结构：【课前准备】→【0-X分钟】小测→讲评→练习→讲评→方法串讲+课后安排
- `**指令：**` = 动作，`你说："..."` = 逐字稿
- 分支判断预判学生具体错误
- 核心原则：**先练后讲**

## 质量检查清单

生成材料后自查：

- [ ] 目录层级正确（4 层）
- [ ] 周次格式正确（两位数字）
- [ ] 科目简称正确（道法不是道德与法治）
- [ ] 学生版不含任何答案/解析
- [ ] 教师版有时间轴 + 逐字稿 + 分支判断
- [ ] 题目围绕下次考试范围
- [ ] 简答题有评分标准
- [ ] 要点总结用表格/框架/口诀

## 红线提醒

- **禁止用 GitHub Contents API 推送文件**（中文会变 0 字节）
- **必须 git clone → 本地编辑 → git commit → git push**
- Materials 分支是 **master**（不是 main）
- Calendar 数据源是 **src/*.md**（不是 schedule.json）
